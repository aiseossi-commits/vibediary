## ADDED Requirements

### Requirement: 태그 항목 롱프레스로 편집 액션 진입
태그 목록의 각 항목은 평상시 수정/삭제 버튼을 SHALL 표시하지 않는다. 사용자가 태그 항목을 길게 누르면 해당 행에 수정(✎)과 삭제(×) 버튼이 SHALL 인라인으로 나타난다.

#### Scenario: 평상시 버튼 미표시
- **WHEN** 태그 목록이 표시될 때
- **THEN** 각 태그 행에 수정/삭제 버튼이 보이지 않는다

#### Scenario: 롱프레스로 버튼 표시
- **WHEN** 사용자가 태그 항목을 길게 누르면
- **THEN** 해당 태그 행 우측에 수정(✎)과 삭제(×) 버튼이 나타난다

#### Scenario: 다른 항목 탭 시 버튼 숨김
- **WHEN** 버튼이 노출된 상태에서 다른 태그를 탭하거나 화면 다른 곳을 누르면
- **THEN** 노출된 수정/삭제 버튼이 사라진다

### Requirement: 인라인 버튼 액션
롱프레스 후 노출된 수정/삭제 버튼은 기존 편집 흐름과 SHALL 동일하게 동작한다.

#### Scenario: 수정 버튼 탭
- **WHEN** 롱프레스 후 수정(✎) 버튼을 탭하면
- **THEN** 해당 태그 행이 인라인 편집 모드로 전환된다

#### Scenario: 삭제 버튼 탭
- **WHEN** 롱프레스 후 삭제(×) 버튼을 탭하면
- **THEN** 기존과 동일한 삭제 확인 Alert가 표시된다
