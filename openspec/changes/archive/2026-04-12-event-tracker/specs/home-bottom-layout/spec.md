## MODIFIED Requirements

### Requirement: 홈 화면 하단 레이아웃

홈 화면 하단 영역은 활성 이벤트 뱃지 행이 최상단, 진주 버튼이 중간, 텍스트 입력창이 아래에 위치해야 한다(MUST). 보내기 버튼은 항상 표시되어야 한다(SHALL). 활성 이벤트가 없을 때는 뱃지 행이 표시되지 않는다.

#### Scenario: 하단 레이아웃 순서 (이벤트 있을 때)
- **WHEN** 홈 화면이 표시되고 활성 이벤트가 존재한다
- **THEN** 이벤트 뱃지 행 → 진주 버튼 → 텍스트 입력창 순서로 표시된다

#### Scenario: 하단 레이아웃 순서 (이벤트 없을 때)
- **WHEN** 홈 화면이 표시되고 활성 이벤트가 없다
- **THEN** 뱃지 행 없이 진주 버튼 → 텍스트 입력창 순서로 표시된다

#### Scenario: 보내기 버튼 비활성 상태
- **WHEN** 텍스트 입력창이 비어 있다
- **THEN** 보내기 버튼이 반투명(35%)으로 표시되며 비활성화된다

#### Scenario: 보내기 버튼 활성 상태
- **WHEN** 텍스트 입력창에 내용이 입력된다
- **THEN** 보내기 버튼이 완전 불투명으로 활성화된다

## ADDED Requirements

### Requirement: 활성 이벤트 뱃지 표시
홈화면 뱃지 행에는 현재 아이의 활성 이벤트가 수평 스크롤 가능한 형태로 표시된다. 각 뱃지는 "이름 · X일째" 형식이며, 탭하면 이벤트 관리 모달이 열린다. "+" 버튼으로 새 이벤트를 추가할 수 있다.

#### Scenario: 뱃지 탭
- **WHEN** 사용자가 활성 이벤트 뱃지를 탭
- **THEN** 이벤트 관리 모달이 열리고 해당 이벤트가 포커스된다

#### Scenario: + 버튼 탭
- **WHEN** 사용자가 "+" 뱃지를 탭
- **THEN** 새 이벤트 추가 모달이 열린다
