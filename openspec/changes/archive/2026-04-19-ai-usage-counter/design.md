## Context

현재 앱은 AI 호출(등대 질문, AI 인사이트 생성)에 아무 제한이 없다. 무료 유저가 헤비하게 사용해도 동일한 API 비용이 발생한다. 유료화 구조를 도입하면서 무료/유료 경계를 만들어야 한다.

현재 관련 파일:
- `src/db/appSettingsDao.ts` — app_settings 테이블 read/write
- `src/db/schema.ts` — DB 마이그레이션 (현재 v16)
- `src/screens/SearchScreen.tsx` — 등대 질문 전송 + 인사이트 생성 버튼

## Goals / Non-Goals

**Goals:**
- 무료 유저 월 AI 호출 10회 제한 (등대 질문 + 인사이트 생성 통합 카운터)
- 월 자동 초기화 (호출 시점에 월이 바뀌었으면 리셋)
- 한도 초과 시 명확한 UX (Alert + 남은 횟수 표시)
- isPremium 플래그 준비 (결제 시스템은 별도 — 지금은 항상 false)

**Non-Goals:**
- 결제/구독 시스템 구현
- 서버 사이드 검증 (클라이언트 카운터로만 운영, 초기)
- 무료 유저 기록 처리(processWithAI) 제한 — 녹음 저장은 무제한 유지

## Decisions

**D1: 카운터 저장 위치 — app_settings DB (로컬)**
- 대안: 서버 사이드 카운터
- 이유: 현재 서버 없음. 로컬로 먼저 구현하고 결제 시스템 도입 시 서버로 마이그레이션. 우회 가능하지만 초기 단계에서 감수할 수준.

**D2: 카운터 훅 분리 — `useAIUsage.ts`**
- 대안: SearchScreen 내부에서 직접 처리
- 이유: 나중에 다른 화면(홈 AI 입력 등)에서도 동일 카운터를 참조해야 함. 훅으로 분리해 단일 진실 공급원 확보.

**D3: 등대 질문 + 인사이트 생성 통합 카운터**
- 대안: 각각 별도 카운터
- 이유: 유저가 직관적으로 "이번 달 AI 7/10 사용"으로 파악하게 통합. 관리도 단순.

**D4: 녹음 처리(processWithAI)는 카운터 미적용**
- 이유: 녹음→저장은 앱 핵심 가치. 제한하면 이탈 유발. 등대/인사이트만 "추가 AI 기능"으로 구분.

## Risks / Trade-offs

- [클라이언트 카운터 우회] 앱 재설치 시 초기화됨 → 초기 단계 감수, 서버 도입 시 해결
- [월 경계 처리] 호출 시점에 월 비교 — 오프라인 상태에서도 정확히 동작
- [isPremium 하드코딩] 현재 false 고정 → 결제 시스템 붙일 때 이 값만 바꾸면 됨
