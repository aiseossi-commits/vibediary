## 웹 리포트 서비스

**배경:** 백업 JSON 파일을 활용해 발달 추이를 시각화하는 웹 서비스.

**구체적으로 고려된 기능들:**

### 병원·치료사용 보고서 자동 생성 (우선순위 높음)
- 백업 JSON 업로드 → 기간 선택 → PDF 리포트 출력
- 포함 내용: 태그별 빈도, 주요 에피소드 타임라인, 투약 요약
- 의사·치료사에게 한 달치 기록을 말로 설명하는 부담을 없애는 것이 목표
- 구현 방식: 정적 HTML(서버 불필요) 또는 Deno Deploy 확장

### 태그 트렌드 차트 (우선순위 높음)
- 기존 태그 데이터만으로 즉시 구현 가능 (AI 재처리 불필요)
- 월별 태그 빈도 막대 그래프 → 치료 효과 측정에 직접 활용
- 요일·시간대별 분포 → 행동 패턴 원인 추정 가능

### 감정·행동 달력 히트맵
- GitHub 잔디 형식으로 날짜별 "힘든 날 / 평범한 날 / 좋은 날" 색상 표시
- 단순 룰 기반(#행동 태그 있으면 주황) 또는 AI 재분석 방식 중 선택 필요

### 수면·체온·투약 트래킹
- `structured_data`에서 체온, 투약 정보 추출 → 시계열 그래프
- 투약 간격 누락 감지 알림 가능
- **선결 조건:** structured_data 필드 표준화 필요 (아래 항목 참고)

## structured_data 필드 표준화

**배경:** 현재 AI가 추출하는 `structured_data`는 자유형식이라 기록마다 필드명이 다름.
그래프나 통계를 의미있게 만들려면 일관된 필드 구조가 필요.

**고려된 표준 스키마:**
```json
{
  "temperature": 38.5,
  "medications": [{ "name": "해열제", "dose": "1정", "time": "09:00" }],
  "behavior_incident": true,
  "behavior_notes": "자해 행동 3회",
  "sleep_hours": 7,
  "meal_status": "양호"
}
```

**영향:** AI 프롬프트 수정 필요. 기존 기록에는 소급 적용 불가.
데이터가 쌓일수록 그래프 품질이 높아지므로 빠를수록 좋음.

**재도입 조건:** 유료 사용자 전용 기능으로 분리하거나, 더 저렴한 모델로 교체 시 재검토 가능.

## 다중 보호자 지원

**배경:** 엄마, 아빠, 조부모 등 여러 보호자가 같은 아이의 기록을 함께 작성하고 공유.

**고려된 구조:**
- 클라우드 동기화 (현재 로컬 전용)
- 기록 작성자 표시
- 열람 권한 관리

**보류 이유:** 서버 구축 비용, 인증 시스템 복잡도, 개인정보 처리 문제.
현재는 백업 파일을 수동으로 공유하는 방식으로 대체.

## 캘린더 날짜 지정 기록 표시 (구현 예정)

**배경:** 캘린더에서 특정 날짜를 선택해 추가한 기록은 시간이 고정(12:00)되어 다른 기록 중간에 끼는 문제.

**결정된 방향:**
- 시간 표시 제거 대신 "추가 기록" 배지 표시
- DB에 `source` 컬럼 추가 (voice / calendar_text 등)
- 정렬은 23:59:59로 고정해 항상 맨 뒤에 위치
