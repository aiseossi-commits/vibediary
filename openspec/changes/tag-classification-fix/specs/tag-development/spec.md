## ADDED Requirements

### Requirement: #발달 기본 태그 존재
`#발달`이 `DEFAULT_TAGS`에 포함되어 신규 아이 생성 시 자동으로 시드되어야 한다.

#### Scenario: 신규 아이 생성 시 #발달 태그 자동 생성
- **WHEN** 사용자가 새 바다(아이)를 생성하면
- **THEN** 해당 아이의 태그 목록에 `#발달`이 포함되어야 한다

#### Scenario: TagsScreen에서 #발달 표시
- **WHEN** 사용자가 TagsScreen을 열면
- **THEN** `#발달` 태그가 목록에 표시되어야 한다

### Requirement: AI가 발달 기록에 #발달 태그 부여
AI는 발달 영역 관찰 또는 성장 이정표 기록에 `#발달` 태그를 부여해야 한다. `#일상` 대신 사용한다.

#### Scenario: 첫 발화 기록에 #발달 태그
- **WHEN** "오늘 처음으로 '엄마'라고 발화했다" 같은 발달 관찰 기록이 입력되면
- **THEN** AI가 `#발달` 태그를 부여해야 한다
- **THEN** `#일상`만 단독으로 부여해서는 안 된다

#### Scenario: 단순 일상 기록에 #발달 미부여
- **WHEN** "오늘 간식 잘 먹었다" 같은 발달과 무관한 기록이 입력되면
- **THEN** AI가 `#발달` 태그를 부여하지 않아야 한다

### Requirement: AI 태그 분류 규칙 명확화
AI 프롬프트는 각 기본 태그의 semantic 정의와 `event_type`과 태그의 독립성을 명시해야 한다.

#### Scenario: #의료 태그 — 단순 체온 관찰 제외
- **WHEN** "체온 재봤는데 37.8도였다, 특별한 처치 없음" 기록이 입력되면
- **THEN** AI가 `#의료` 태그를 부여하지 않아야 한다

#### Scenario: #의료 태그 — 투약 포함 시 부여
- **WHEN** "체온 38.5도, 타이레놀 5ml 복용" 기록이 입력되면
- **THEN** AI가 `#의료`와 `#투약` 태그를 부여해야 한다

#### Scenario: 커스텀 태그 추가 적용
- **WHEN** 사용자가 `#수면문제` 커스텀 태그를 생성했고, "밤새 잠을 못 잤다" 기록이 입력되면
- **THEN** AI가 `#수면문제` 태그를 추가로 부여해야 한다
- **THEN** 기본 태그(`#일상` 등)와 함께 병존해야 한다
