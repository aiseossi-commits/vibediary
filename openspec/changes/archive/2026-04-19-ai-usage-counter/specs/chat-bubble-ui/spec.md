## MODIFIED Requirements

### Requirement: 슬라이딩 윈도우 대화 관리
대화 히스토리는 세션 메모리에만 유지되며, LLM 호출 시 최근 2교환(사용자+AI 메시지 각 2개)만 컨텍스트로 포함한다. **질문 전송 전 AI 사용 카운터를 확인하고, 무료 한도 초과 시 LLM 호출을 차단한다.**

#### Scenario: 첫 번째 질문
- **WHEN** 세션 첫 질문을 전송하면
- **THEN** 이전 대화 없이 현재 질문과 검색된 기록만으로 LLM을 호출한다

#### Scenario: 후속 질문 컨텍스트
- **WHEN** 3번째 이상 질문을 전송하면
- **THEN** 가장 최근 2교환(메시지 4개)만 LLM 컨텍스트에 포함하고, 이전 대화는 제외한다

#### Scenario: 세션 초기화
- **WHEN** 사용자가 앱을 재시작하거나 탭을 나갔다 돌아오면
- **THEN** 대화 히스토리가 초기화되고 빈 상태 화면이 표시된다

#### Scenario: 한도 초과 시 전송 차단
- **WHEN** 무료 유저의 `ai_usage_count`가 10 이상인 상태에서 전송 버튼을 누르면
- **THEN** LLM을 호출하지 않고 업그레이드 유도 Alert를 표시한다
