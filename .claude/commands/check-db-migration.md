# DB 마이그레이션 안전 검사

`src/db/schema.ts`와 `src/db/database.ts`를 읽고, 최근 추가된 DB 변경이 안전한지 검사한다.

## 검사 항목

1. **마이그레이션 번호 연속성** — database.ts의 마이그레이션 버전이 순서대로 빠짐없이 이어지는가?
2. **트랜잭션 래핑** — 다중 테이블을 건드리는 마이그레이션에 트랜잭션이 적용되어 있는가?
3. **DAO 동기화** — schema.ts에 새로 추가된 컬럼/테이블이 대응하는 DAO 파일에 반영되어 있는가?
4. **index.ts 익스포트** — 새 DAO가 `src/db/index.ts`에 익스포트되어 있는가?
5. **NULL 안전성** — 기존 데이터가 있는 테이블에 NOT NULL 컬럼을 추가할 때 DEFAULT 값이 있는가?
6. **중복 마이그레이션** — 동일한 ALTER TABLE이 두 번 실행될 경우 try-catch로 보호되어 있는가?

## 출력 형식

```
현재 DB 버전: vN
최근 변경 테이블: ...

✅ 통과: ...
⚠️  주의: ...
❌ 위험: ...
```

위험 항목이 있으면 구체적인 수정 방법도 함께 제시한다.
