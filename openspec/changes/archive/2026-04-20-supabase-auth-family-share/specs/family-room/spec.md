## ADDED Requirements

### Requirement: 가족방 생성 및 초대코드 발급
인증된 사용자는 가족방을 생성할 수 있어야 하며(SHALL), 생성 시 6자리 대문자+숫자 형식의 고유 초대코드가 자동 발급되어야 한다(SHALL).

#### Scenario: 가족방 생성
- **WHEN** 사용자가 "가족방 만들기" 버튼을 탭할 때
- **THEN** Supabase `families` 테이블에 새 레코드를 생성하고, `family_members`에 생성자를 등록하며, 6자리 초대코드를 화면에 표시한다

#### Scenario: 초대코드 중복 시 재생성
- **WHEN** 생성된 초대코드가 기존 코드와 충돌할 때
- **THEN** 새 코드를 자동으로 재생성하여 유일성을 보장한다

#### Scenario: 초대코드 복사
- **WHEN** 사용자가 초대코드 옆 복사 버튼을 탭할 때
- **THEN** 코드가 클립보드에 복사되고 "복사됨" 피드백을 표시한다

### Requirement: 초대코드로 가족방 참여
사용자는 6자리 초대코드를 입력하여 기존 가족방에 참여할 수 있어야 한다(SHALL). 참여 후 같은 가족방의 모든 아이 데이터에 접근할 수 있어야 한다(SHALL).

#### Scenario: 유효한 코드로 참여
- **WHEN** 사용자가 유효한 6자리 초대코드를 입력하고 확인 버튼을 탭할 때
- **THEN** `family_members`에 사용자를 추가하고 가족방 참여 성공 메시지를 표시한다

#### Scenario: 잘못된 코드 입력
- **WHEN** 존재하지 않는 초대코드를 입력할 때
- **THEN** "유효하지 않은 초대코드입니다" 오류 메시지를 표시하고 방에 참여하지 않는다

#### Scenario: 이미 참여 중인 방에 재참여 시도
- **WHEN** 이미 속해 있는 가족방의 초대코드를 입력할 때
- **THEN** "이미 참여 중인 가족방입니다" 메시지를 표시한다

### Requirement: 멤버 수 제한 없는 가족방 참여
가족방은 엄마, 아빠, 조부모, 치료사, 교사 등 멤버 수 제한 없이 참여 가능해야 한다(SHALL).

#### Scenario: 3인 이상 참여
- **WHEN** 3명 이상의 사용자가 같은 초대코드로 순차 참여할 때
- **THEN** 모든 멤버가 `family_members`에 등록되고 가족방에 정상 접근한다

### Requirement: 가족 공유 UI (설정 화면)
설정 화면에 "가족 공유" 섹션이 있어야 하며(SHALL), 여기서 방 생성, 코드 입력, 현재 참여 상태를 확인할 수 있어야 한다(SHALL).

#### Scenario: 가족 공유 미설정 상태
- **WHEN** 사용자가 가족 공유에 참여하지 않은 상태에서 설정을 열 때
- **THEN** "가족방 만들기"와 "초대코드로 참여" 두 가지 옵션을 표시한다

#### Scenario: 가족방 참여 상태 확인
- **WHEN** 사용자가 가족방에 참여한 상태에서 설정을 열 때
- **THEN** 현재 초대코드, 멤버 수를 표시하고 "가족방 나가기" 옵션을 제공한다
