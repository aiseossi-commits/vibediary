## ADDED Requirements

### Requirement: 이벤트 생성
사용자는 기본 목록에서 선택하거나 직접 입력하여 이벤트를 생성할 수 있다. 시작일은 기본값 오늘이며 과거 날짜로 변경 가능하다. 생성 즉시 활성 상태가 된다.

#### Scenario: 기본 목록에서 이벤트 생성
- **WHEN** 사용자가 기본 이벤트 목록에서 항목(예: "감기")을 탭하고 확인
- **THEN** 해당 이벤트가 오늘 시작일로 DB에 저장되고 홈화면 뱃지에 즉시 표시된다

#### Scenario: 커스텀 이벤트 생성
- **WHEN** 사용자가 직접 이름을 입력하고 시작일을 선택한 후 저장
- **THEN** 해당 이름과 시작일로 이벤트가 생성된다

#### Scenario: 과거 시작일 지정
- **WHEN** 사용자가 시작일을 3일 전으로 설정
- **THEN** 뱃지에 "4일째"로 표시된다

### Requirement: 기간 표시 자동 변환
시스템은 started_at으로부터 현재까지의 경과일을 계산하여 단위를 자동 변환한다.

#### Scenario: 7일 미만
- **WHEN** 이벤트 시작 후 경과일이 6일
- **THEN** 뱃지에 "6일째"로 표시된다

#### Scenario: 7일 이상 28일 미만
- **WHEN** 경과일이 14일
- **THEN** 뱃지에 "2주째"로 표시된다

#### Scenario: 28일 이상
- **WHEN** 경과일이 90일
- **THEN** 뱃지에 "3개월째"로 표시된다

### Requirement: 이벤트 종료
사용자는 활성 이벤트를 수동으로 종료할 수 있다. 종료일은 오늘 또는 과거 날짜를 선택할 수 있다.

#### Scenario: 오늘 종료
- **WHEN** 사용자가 이벤트의 "종료" 버튼을 탭하고 "오늘 종료" 선택
- **THEN** ended_at이 현재 시각으로 저장되고 홈화면 뱃지에서 제거된다

#### Scenario: 과거 날짜로 종료
- **WHEN** 사용자가 종료일을 어제로 선택
- **THEN** ended_at이 해당 날짜 23:59:59로 저장된다

### Requirement: 여러 이벤트 동시 활성
시스템은 동일한 아이에 대해 여러 이벤트를 동시에 활성 상태로 유지할 수 있다.

#### Scenario: 두 이벤트 동시 활성
- **WHEN** "감기"와 "변비" 이벤트가 모두 활성 상태
- **THEN** 홈화면에 두 뱃지가 모두 표시된다

### Requirement: 아이별 이벤트 분리
이벤트는 child_id 기준으로 분리된다.

#### Scenario: 아이 전환 시 이벤트 변경
- **WHEN** 활성 아이를 전환
- **THEN** 홈화면 뱃지가 해당 아이의 이벤트만 표시한다
