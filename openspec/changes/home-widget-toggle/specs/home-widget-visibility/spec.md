## ADDED Requirements

### Requirement: 위젯 표시 설정 저장
앱은 홈화면 위젯별 표시 여부를 `app_settings` SQLite 테이블에 저장해야 한다(SHALL). 설정되지 않은 위젯은 기본값 ON으로 처리해야 한다(MUST). 모든 위젯의 기본값은 ON이다(MUST).

#### Scenario: 신규 설치 기본값
- **WHEN** 신규 설치 후 앱을 처음 실행한다
- **THEN** 모든 위젯(음성 입력, 텍스트 입력, 증상 추적, 최근 기록)이 ON 상태로 표시된다

#### Scenario: 설정 저장 및 즉시 반영
- **WHEN** 사용자가 설정 탭에서 특정 위젯 토글을 OFF로 변경한다
- **THEN** 해당 위젯이 즉시 홈화면에서 사라지며 DB에 OFF 상태로 저장된다

#### Scenario: 앱 재시작 후 유지
- **WHEN** 위젯을 OFF로 설정 후 앱을 재시작한다
- **THEN** 해당 위젯이 OFF 상태로 유지된다

### Requirement: 설정 탭 홈화면 구성 섹션
설정 화면에 "홈화면 구성" 섹션이 표시되어야 한다(SHALL). 4개 위젯 각각의 이름과 토글 스위치가 표시되어야 한다(MUST).

#### Scenario: 위젯 목록 표시
- **WHEN** 설정 탭을 열면
- **THEN** "홈화면 구성" 섹션에 음성 입력·텍스트 입력·증상 추적·최근 기록 항목과 토글이 표시된다

#### Scenario: 현재 상태 반영
- **WHEN** 설정 탭을 열면
- **THEN** 각 토글이 현재 저장된 위젯 표시 상태를 반영한다

### Requirement: 홈화면 조건부 렌더링
홈화면은 위젯 설정이 ON인 섹션만 렌더링해야 한다(MUST).

#### Scenario: 위젯 OFF 시 미표시
- **WHEN** 특정 위젯이 OFF 상태다
- **THEN** 해당 섹션이 홈화면에 렌더링되지 않는다

#### Scenario: 음성 입력 OFF
- **WHEN** 음성 입력 위젯이 OFF 상태다
- **THEN** 진주 버튼(녹음 버튼)이 홈화면에 표시되지 않는다

#### Scenario: 텍스트 입력 OFF
- **WHEN** 텍스트 입력 위젯이 OFF 상태다
- **THEN** 텍스트 입력창과 전송 버튼이 홈화면에 표시되지 않는다

#### Scenario: 최근 기록 OFF
- **WHEN** 최근 기록 위젯이 OFF 상태다
- **THEN** 기록 카드 피드가 홈화면에 렌더링되지 않는다
