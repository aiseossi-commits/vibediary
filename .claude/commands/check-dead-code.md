# 데드코드 탐지

`knip`을 실행해서 사용되지 않는 파일, exports, 의존성을 탐지한다.

## 실행

```bash
npx knip
```

## 결과 해석

knip 결과를 아래 기준으로 분류해서 보고한다.

- **Unused files** — 어디서도 import되지 않는 파일. 삭제 후보.
- **Unused exports** — export됐지만 다른 파일에서 쓰이지 않는 함수/타입. 제거 후보.
- **Unused dependencies** — package.json에 있지만 코드에서 쓰이지 않는 패키지. 제거 후보.
- **Unlisted dependencies** — 코드에서 쓰이지만 package.json에 없는 패키지. 추가 필요.

## 출력 형식

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
데드코드 탐지 결과
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ 이상 없음  또는

🗑️  삭제 후보 파일: N개
   - 파일명: 이유

📤 미사용 exports: N개
   - 파일명: export명

📦 미사용 패키지: N개
   - 패키지명

⚠️  누락 패키지: N개
   - 패키지명 (사용 위치)

권장 조치:
1. ...
2. ...
```

결과가 없으면 "데드코드 없음 ✓"로 끝낸다.
knip 결과에 대해 실제로 삭제해도 되는지 3원칙(안전성 > 친화성 > 효율성) 기준으로 판단 의견도 함께 제시한다.
