# chat-bubble-ui

## Purpose

AI 등대 화면의 대화형 채팅 버블 UI. 외부 채팅 라이브러리 없이 직접 구현하며, 슬라이딩 윈도우 방식으로 대화 히스토리를 관리한다.

---

## Requirements

### Requirement: 채팅 버블 레이아웃
AI 등대는 단발 Q&A 대신 대화형 버블 UI로 표시되어야 한다. 외부 채팅 라이브러리를 사용하지 않고 직접 구현한다.

#### Scenario: 사용자 메시지 표시
- **WHEN** 사용자가 질문을 전송하면
- **THEN** 화면 오른쪽에 primary 배경색의 말풍선으로 표시된다

#### Scenario: AI 답변 표시
- **WHEN** AI 답변이 도착하면
- **THEN** 화면 왼쪽에 surface 배경색의 말풍선으로 표시되고, Reanimated FadeInDown 애니메이션으로 등장한다

#### Scenario: 근거 기록 접기/펼치기
- **WHEN** AI 답변에 근거 기록이 있으면
- **THEN** 답변 버블 아래 "근거 N건" 토글 버튼이 표시되며 기본 상태는 접힘이다
- **WHEN** 사용자가 토글 버튼을 누르면
- **THEN** 근거 RecordCard 목록이 펼쳐지거나 접힌다

---

### Requirement: 슬라이딩 윈도우 대화 관리
대화 히스토리는 세션 메모리에만 유지되며, LLM 호출 시 최근 2교환(사용자+AI 메시지 각 2개)만 컨텍스트로 포함한다.

#### Scenario: 첫 번째 질문
- **WHEN** 세션 첫 질문을 전송하면
- **THEN** 이전 대화 없이 현재 질문과 검색된 기록만으로 LLM을 호출한다

#### Scenario: 후속 질문 컨텍스트
- **WHEN** 3번째 이상 질문을 전송하면
- **THEN** 가장 최근 2교환(메시지 4개)만 LLM 컨텍스트에 포함하고, 이전 대화는 제외한다

#### Scenario: 세션 초기화
- **WHEN** 사용자가 앱을 재시작하거나 탭을 나갔다 돌아오면
- **THEN** 대화 히스토리가 초기화되고 빈 상태 화면이 표시된다
