---

기능을 추가, 제거, 보완, 수정할 때 반드시 아래 세 원칙을 점검한다.

## 1. 앱 안전성
- 기존 기능/데이터에 영향 없는가
- 에러 처리, 데이터 손실 위험 있는가
- TypeScript 타입 체크 통과하는가

## 2. 유저 친화성
- 돌봄 가족 입장에서 직관적인가
- 진입점이 자연스러운가
- 불필요한 단계가 없는가

## 3. 토큰 효율성
- 불필요한 API 호출이 생기는가
- 컨텍스트를 최소화했는가
- 오프라인에서도 동작 가능한가

**Why:** 앱의 핵심 사용자(발달장애인 돌봄 가족)를 위한 품질 기준. 기능 구현 전 설계 단계에서 점검.

**How to apply:** 새 기능 제안/구현 시작 전, opsx:new 진입 전에 이 체크리스트로 먼저 점검. 추후 스킬화 예정.
